# O*NET Data Warehouse Prototype

---

## Overview

This project is a lightweight data warehouse prototype built on O*NET occupation data.  
It is designed to support workforce program dashboards and labor market analysis for Tulsa.

**Stack:** Python 3 · pandas · SQLite · SQL  
**No cloud dependencies. Runs entirely locally.**

---

## Dataset

**Source:** [O*NET OnLine Database](https://www.onetcenter.org/database.html)  
**Focus tables:** Occupation Data, Skills, Knowledge, Education/Training/Experience, Task Statements, Wages

The O*NET database is the US Department of Labor's primary taxonomy of occupational information.  
It provides standardized, occupational-level data on hundreds of job characteristics — skills, knowledge,  
education requirements, tasks, and more — across ~1,000 SOC-coded occupations.

This prototype uses the real O*NET database files alongside BLS wage data for accurate, production-representative results.

## Schema Overview

The warehouse follows a **star schema** pattern with one central dimension and five fact tables.

```
                        ┌──────────────────────┐
                        │   dim_occupation      │
                        │──────────────────────│
                        │ onetsoc_code (PK)     │
                        │ title                 │
                        │ description           │
                        │ soc_major_group       │
                        └──────────┬───────────┘
               ┌──────────────────┼──────────────────────┐
               │                  │                       │
    ┌──────────▼──────┐ ┌─────────▼────────┐ ┌──────────▼──────┐
    │  fact_skills    │ │  fact_knowledge  │ │  fact_education │
    │─────────────────│ │──────────────────│ │─────────────────│
    │ onetsoc_code FK │ │ onetsoc_code FK  │ │ onetsoc_code FK │
    │ element_id      │ │ element_id       │ │ element_id      │
    │ element_name    │ │ element_name     │ │ element_name    │
    │ scale_id        │ │ scale_id         │ │ scale_id        │
    │ data_value      │ │ data_value       │ │ category        │
    │ date_updated    │ │ date_updated     │ │ data_value      │
    └─────────────────┘ └──────────────────┘ └─────────────────┘
               │                  │
    ┌──────────▼──────┐ ┌─────────▼────────┐
    │   fact_tasks    │ │   fact_wages     │
    │─────────────────│ │──────────────────│
    │ onetsoc_code FK │ │ onetsoc_code FK  │
    │ task_id         │ │ annual_median    │
    │ task            │ │ annual_10th_pct  │
    │ task_type       │ │ annual_90th_pct  │
    │ incumbents_resp │ │ hourly_median    │
    └─────────────────┘ │ wage_year        │
                        └──────────────────┘
```

### Design Decisions

- **Star schema** over normalized snowflake: simpler queries for analysts, easier BI tool integration.
- **`soc_major_group`** derived column added to `dim_occupation` to enable group-level aggregations without a join.
- **`scale_id`** retained in fact tables (`IM` = Importance, `LV` = Level) to allow filtering without extra dimension tables.
- **SQLite** used as the warehouse engine — zero-config, file-based, fully SQL-compliant. Easy to migrate to DuckDB, Postgres, or Snowflake later.

## ETL Logic

### Files

| File | Purpose |
|---|---|
| `etl_pipeline.py` | Full ETL: Extract → Transform → Load → Validate |
| `run_queries.py` | Runs analytical queries and prints results |
| `queries.sql` | Raw SQL query file (6 annotated queries) |
| `warehouse.db` | Output SQLite database (generated by pipeline) |

### Extract
Reads CSV files from `raw_data/` using pandas. Each file maps to one O*NET source table.

### Transform
Each table has a dedicated transform function that:
- **Normalizes column names** (lowercase, underscores, removes special characters like `*` and `-`)
- **Drops null key rows** (e.g., rows missing SOC code or title)
- **Normalizes SOC codes** (uppercase, stripped whitespace)
- **Clips numeric scales** to valid O*NET ranges (skills/knowledge: 0–7, education: 0–100)
- **Derives fields** (e.g., `soc_major_group` from first 2 digits of SOC code)
- **Coerces data types** (dates to datetime, numeric columns to float/int)

### Load
Transformed DataFrames are loaded into SQLite via `pandas.DataFrame.to_sql()`.  
Tables are replaced on each run (`if_exists='replace'`) for idempotency.

---

## Data Validation

Six checks run automatically at the end of the pipeline:

| Check | What It Catches |
|---|---|
| No null SOC codes in `dim_occupation` | Missing primary keys |
| No null titles in `dim_occupation` | Missing required dimension attributes |
| Skills `data_value` in [0, 7] | Out-of-range scale scores |
| Knowledge `data_value` in [0, 7] | Out-of-range scale scores |
| Wages non-negative | Invalid wage values |
| No orphaned skill records | Referential integrity (FK violations) |

All checks passed on the current dataset ✅

## How to Run

### Requirements
```bash
python3 --version   # 3.9+
pip install pandas  # only external dependency
```

### Steps
```bash
# 1. Install dependencies
pip3 install pandas openpyxl

# 2. Run ETL pipeline (creates warehouse.db)
python3 etl_pipeline.py

# 3. Run analytical queries
python3 run_queries.py

# Optional: open the database directly
sqlite3 warehouse.db
```

### Data Sources
- **O*NET data**: Downloaded from https://www.onetcenter.org/database.html
  - Occupation Data.xlsx
  - Skills.xlsx
  - Knowledge.xlsx
  - Education, Training, and Experience.xlsx
  - Task Statements.xlsx
- **Wage data**: Downloaded from https://www.bls.gov/oes/tables.htm
  - National occupational employment and wage estimates (wages.xlsx)
